1. INTRODUCTION                                   *ai-gitcommit-intro*

ai-gitcommit.nvim generates commit messages from staged git diff.

Supported providers:
  - OpenAI (and compatible endpoints)
  - GitHub Copilot

Requirements:
  - Neovim 0.11+
  - curl

==============================================================================
2. CONFIGURATION                                  *ai-gitcommit-config*

IMPORTANT:
Old flat config is not supported anymore. Use `provider` + `providers`.
`provider` defaults to "copilot"; if copilot.vim or copilot.lua is installed
and authenticated, no configuration is needed.

Minimal setup (Copilot, default): >
  require("ai-gitcommit").setup()

Minimal setup (OpenAI): >
  require("ai-gitcommit").setup({
    provider = "openai",
    providers = {
      openai = {
        api_key = vim.env.OPENAI_API_KEY,
      },
    },
  })

Full setup: >
  require("ai-gitcommit").setup({
    provider = "copilot", -- "openai" | "copilot" (default: "copilot")

    providers = {
      openai = {
        api_key = vim.env.OPENAI_API_KEY,
        api_key_required = true,
        api_key_header = "Authorization",
        api_key_prefix = "Bearer ",
        extra_headers = {},
        stream_options = true,
        model = "gpt-4o-mini",
        endpoint = "https://api.openai.com/v1/chat/completions",
        max_tokens = 500,
      },

      copilot = {
        -- model = nil → auto-pick cheapest available from /models
        model = nil,
        endpoint = "https://api.githubcopilot.com/chat/completions",
        max_tokens = 500,
      },
    },

    languages = { "English", "Chinese", "Japanese", "Korean" },
    prompt_template = nil,

    context = {
      max_diff_lines = 500,
      max_diff_chars = 15000,
    },

    filter = {
      exclude_patterns = {
        "%.lock$",
        "package%-lock%.json$",
        "yarn%.lock$",
        "pnpm%-lock%.yaml$",
        "%.min%.[jc]ss?$",
        "%.map$",
      },
      exclude_paths = {}, -- path rules (Lua patterns)
      include_only = nil, -- only include matching paths
    },

    keymap = nil,
    auto = {
      enabled = true,
      debounce_ms = 450,
    },
  })

Custom prompt placeholders:
  - {language}
  - {extra_context}
  - {staged_files}
  - {diff}

Diff context behavior:
  - Files matching `exclude_patterns` are removed.
  - Files matching `exclude_paths` are removed.
  - If `include_only` is non-empty, only matching files are included.
  - Context is truncated by `context.max_diff_lines`, then by
    `context.max_diff_chars`.

Provider config validation:
  - providers.<name>.model must be non-empty string
  - providers.<name>.endpoint must be non-empty string
  - providers.<name>.max_tokens must be > 0

3. COMMANDS                                       *ai-gitcommit-commands*

:AICommit [{context}]                              *:AICommit*
  Generate commit message in gitcommit buffer.

:AICommit logout <provider>                       *:AICommit-logout*
  Logout provider.

:AICommit status                                  *:AICommit-status*
  Show all provider status.

:AICommit status <provider>                       *:AICommit-status-provider*
  Show one provider status.

`<provider>` must be one of:
  - openai
  - copilot

Notes:
  - OpenAI uses API key config.
  - Copilot reads tokens from copilot.vim or copilot.lua.
  - Empty provider output is reported as warning (no false success message).
  - Git command failures are reported as errors.

4. PROVIDERS                                     *ai-gitcommit-providers*

OpenAI:
  Configure `providers.openai.api_key`.

OpenAI-compatible:
  Reuse the `openai` provider and customize:
    - `providers.openai.endpoint`
    - `providers.openai.api_key_required = false` (for no-auth endpoints)
    - `providers.openai.api_key_header` and `api_key_prefix`
    - `providers.openai.extra_headers`
    - `providers.openai.stream_options = false` (if needed)
    - Streaming parser accepts LF and CRLF SSE line endings

Copilot:
  Reads OAuth tokens from an installed Copilot plugin:
    - copilot.vim (run `:Copilot auth`)
    - copilot.lua
  No separate login required for this plugin.

  Model selection:
    If `providers.copilot.model` is unset (nil), the plugin fetches the list
    of chat models your subscription allows via Copilot's /models endpoint
    and picks the one with the lowest `billing.multiplier` (cheapest). Set
    an explicit model ID to override. The resolved list is cached for
    30 minutes.

==============================================================================
5. TROUBLESHOOTING                               *ai-gitcommit-troubleshooting*

"No provider configured"
  `provider` defaults to "copilot". If you set it to "" explicitly, restore
  the default or set `provider = "openai"|"copilot"`.

"OpenAI API key not configured"
  Set `providers.openai.api_key`.

"Not authenticated"
  For Copilot, install copilot.vim or copilot.lua and authenticate there.

"Not in a gitcommit buffer"
  The command works only in commit message buffer (opened by `git commit`).
